## 💡 ЧО МИ ЗРОБИЛИ ЦІЄЇ СЕСІЇ

### Проблема
Undo/Redo система зберігає ПОВНІ СНІМКИ всього проекту (~100 KB на операцію).  
Це спричиняє швидкий ріст RAM: 50 операцій × 100 KB = **~5 MB памяті**

### Рішення
Замість снімків, команди зберігають **тільки дельта** (зміни):
```
AddShapeCommand:        ~1 KB      (vs 100 KB snapshot = 100x меньше)
DeleteShapeCommand:     ~1-2 KB    (vs 100 KB snapshot = 50x меньше)
TransformCommand:       ~200 bytes (vs 100 KB snapshot = 500x меньше!) ⭐
PropertyChangeCommand:  ~500 bytes (vs 100 KB snapshot = 200x меньше)

Результат: 50 операцій × 200 байт = **~10 KB памяті**
Чистий результат: 5 MB → 10 KB = **500x меньше памяті!**
```

### 2️⃣ **Гібридна система Undo** ✅  
Файл: `js/cad/core/undo.js` (змінено)

**Ключова ідея:** Обидва способи роблять одночасно!

```javascript
// ДО (snapshot - все ще працює):
saveState('Move 5 objects')  // Зберігає 100 KB

// ПІСЛЯ (команди - в сертifies):
registerCommand(new TransformCommand(...))  // Зберігає 200 bytes

// Обидва способи можуть бути в одному стеку!
// Система автоматично выявляет тип і виконує потрібну логіку
```

**Все залишається сумісним!** Користувач нічого не помітить.
